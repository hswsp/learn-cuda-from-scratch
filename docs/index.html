CUDA × LLM 推理 实战教程

📚 14 章 💻 60+ 个示例 🐍 中文 · CUDA C++ 🎯 目标：手写 GPT-2 推理

这是一份从零起步的 CUDA GPU 编程教程，目的不是让你"用 PyTorch 调几行就跑通模型"， 而是让你真正理解大模型推理是怎么在硬件上跑起来的， 最终亲手写出一个能跑 GPT-2 small 的推理引擎。

📍 学习准备： 你需要会基础 C/C++，不需要任何并行编程经验； 不需要在本机装 CUDA Toolkit，用 Google Colab 即可一键运行。

快速开始

方式 1: Google Colab（推荐初学者）

无需本地 GPU，colab.research.google.com → Runtime → Change runtime type → T4 GPU → 打开仓库里的 scripts/setup_colab.ipynb。

方式 2: 本地 Linux + NVIDIA GPU

git clone https://github.com/YOUR/ops.git && cd ops
make ARCH=sm_80 -C code/ch02_hello run        # A100/3090
make ARCH=sm_75 -C code/ch02_hello run        # T4

方式 3: 本机 macOS（只学不跑 kernel）

python3 -m http.server -d docs 8000           # 预览教程
clang++ -O2 -std=c++17 -DCPU_ONLY \
    code/common/cpu_ref_demo.cpp -o /tmp/ref && /tmp/ref   # CPU 端正确性验证

这个教程的特点

  • 每个 kernel 都有 CPU reference 对拍—— 不靠运气写代码
  • 从 deviceQuery 到 FlashAttention 一条龙—— 没有中间断层
  • 讲性能也讲为什么—— 用 Nsight 看 roofline，不只看 ms
  • 面向真实推理栈—— Ch12-14 直接对标 vLLM / TensorRT-LLM 核心算子

常见问题

Q: 我没有 NVIDIA 显卡，能学吗？

能。所有 HTML 教程在 macOS / Windows / Linux 上都能读； CPU reference 实现可在任何机器上验证算法正确性； 真正想跑 kernel，免费的 Google Colab T4 完全够用（Capstone GPT-2 small 都能跑）。

Q: 已经会用 PyTorch，还需要学 CUDA 吗？

如果你只是用模型，不需要。如果你想： ① 优化推理延迟、② 理解 vLLM/TensorRT-LLM 源码、③ 写自定义算子、④ 准备 GPU 方向面试， 那 CUDA 是绕不过去的台阶。

Q: Triton/CUTLASS 这些不是更香吗？

是的，但是要先理解 CUDA 编程模型才能用好它们。本教程在第 12 章会展示 Triton 对照实现， 第 9 章会对照 CUTLASS，让你看清"高层抽象"背后到底在做什么。

Q: 学完能达到什么水平？

能独立写出 100~1000 行级别的 CUDA kernel，能读懂主流推理引擎源码， 能用 Nsight 定位性能瓶颈。距离"写出能上线的高性能 kernel"还有差距，但这是必要的起点。
